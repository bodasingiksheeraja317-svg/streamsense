Purpose & Confidentiality You are joining Project STREAMSENSE, an OSL research effort to build a modality-agnostic engine that classifies one-dimensional sampled-signal streams into a fixed set of categories. For this sprint the engine is software-only and runs on standard systems — no embedded hardware is required. Data reaches the engine over Ethernet (TCP/IP): one system streams a 1-D sample stream and a second, high-end system processes it at a fast rate. Your work package, Track A, is the host-side half: on a workstation you build the data pipeline, the analysis, the neural-network model, and the training / quantization / packaging flow that produces a small, deployable model. The engine is validated on an openly available audio dataset; audio is the lab validation modality. The engine is deliberately built to be modality-agnostic, so you must not hard-code any audiospecific assumption into the architecture beyond the agreed front-end transform. A clean package is produced so the same model can later be deployed to embedded hardware (see the shared Deployment Package, Section 12 and 8). NEED-TO-KNOW  The end application, the deployment site, and the programme timeline beyond this work package are intentionally not disclosed and are not required to complete your task.
